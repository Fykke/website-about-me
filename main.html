Dodge


My favorite project is my game called "Dodge".
I wrote the first version of it around February 10, 2021.
Finally completed the project on March 7th.


About the game process:

The point of the game is that the player playing as the square needs to dodge the balls flying to the left from the right frame. The square is controlled by pressind WASD keys.
Every 5 points, the balls slightly increase their speed, which complicates the dodging process.

Player can't escape the playing field:
  • If player passes the frame from above, he will be moved down in the opposite direction
  • The same happens if the player goes up
  • Going to the left, the player crashes into the frame


Other:

The main menu has settings and a leaderboard.
In the settings, player can customize the sound, interface and audio.
In the "Theme" section player can change game interface colors. There are 4 options.
In the "Audio" section player can change the volume. Is scaled from 1 to 10.
The "Skin" section, as the name suggests, gives the opportunity to change the player's skin.

Applying all settings does not require restarting the game.

After the game is openned, it randomly decides what skin will be used. It won't be changed after death.
The skin of the ball will change randomly after every lose. (two of the ball's skin are my friend's face 😈. Other skins are the ones that I made in Photoshop using my imagination or redrawn by me items from Minecraft)

After player collides with the ball:

The game will show:
Time alive (seconds), score (how many times the ball reached left frame), and a few options:
  • Play again
  • To the menu
  • Save score (requires to type player's name)


Leaderboard:

Actually, it is not correct to name it "scoreboard", because it is "leaderboard". Only after i had done this part of the page I noticed this mistake in the game. And it's kind of late to make any changes. I can, but don't want to.
So, about the leaderboard. It reads the json file and simply prints all the data from the file. And sorts it by value i.e. collected points (descending).


What have been used:

I used my favourite language Python.
All the logic in my game is made on this language.
All the visual part of the game as well as the sounds are made on pygame.
Also to work with data I imported json, os and sys.
To provide random ball position to fly from, I used randint and choice from module random.
To create an exe file have been used pyinstaller.
I used a lot of loops, functions and also used try/catch constructions to work with files.

Also I created an anti AFK system:
If a player decides to stay on one place to get a lot of points, after approximately 3 seconds the game will throw a ball right at him and he will lose.

Github link: https://github.com/Fykke/Dodge

Enjoy the game! ♥

Excel data comparing


My second (order not based on project creation date), more useful project, is a program that helps my mom with her work.
The latest version of the project was created around June 28, 2021.
Writing, if I'm not mistaken, took a couple of days, although the program is quite easy, since I had absolutely no idea how to work with Excel using code.

With this project I really helped my mother a lot.
Now, instead of losing a few days on reading nad comparing numbers, my program is doing it in a seconds.



What has to be done:

Mom has tons of data at work. Sales column and purchases column (contain the exact price of the item). She has to compare these two columns, and the ideal result is a complete match of both columns. But this never happens, because someone constantly makes mistakes in filling out, or data on any purchases and sales simply do not come due to various reasons.

After comparing the tables (usually contains about 750 sales and 750 purchases), always 5 to 20 inequalities are revealed. Then she needs to figure out what these inequalities are, and why they appeared.


What my program is doing:

First of all, it is a console program and after opening an exe file, it will print to the console an example of what the user should write in.
User has to enter the name of the .xmls file that exists in the same directory as the program exe file. If it doesn't exist, the program will stop working and will be closed.

Also, in the excel file all the data should be written in the first 2 columns (A and B), if it is not, program will not work correctly.

If file exists and everything is fine:

The program will:
  • Print "Updated", in the next line elapsed time since the start of the program and will close itself
  • Then it will update an existing Excel file
  • In the D column it will print all the data that exist in the A column, but did not find a pair (the same number) in the B column
  • In the E column it will print all the data that exist in the B column, but did not find a pair in the A column



What have been used:

This time i also worked with Python.
To create an exe file have been used pyinstaller.
To open and rewrite an Excel file I used openpyxl module and imported os to see computer's files. Also, I wanted it to have at least some kind of beauty, I imported time, to print elapsed time in the console as well as Font, Color, Border, colors, Side from openpyxl.
And of course the data comparing. It is done with the help Counter imported from Collections.

In these type of programs is better to use a C languages, because they are much faster.
But in that moment i knew only Python and also it's not an actual product so there is no need to make it as fast as possible.

You can try it yourself! ↴

Github link: https://github.com/Fykke/Excel-data-comparing
